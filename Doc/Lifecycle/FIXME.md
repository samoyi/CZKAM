# FIXME List

## 使用方法
 * 从一开始就建立独立的 FIXME List ，每一项 FIXME 都进行记录
 * 在正式上线前检查 FIXME List，保证上面没有影响使用的项目。
 * 修改的项目不能删除，只能标记完成


## 优先级管理
* 错误：功能无法正常使用
* 安全：存在安全漏洞
* 视觉：影响视觉体验，但不影响功能，同时不明显影响使用效率
* 效率：产品设计逻辑不良、视觉设计混乱以及程序效率低下导致的用户使用效率降低
* 维护：增加维护成本,增加错误风险
